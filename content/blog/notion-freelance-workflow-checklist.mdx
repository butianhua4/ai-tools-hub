# Notion 管理接单流程怎么用：新手检查清单

这份清单用于搭建 Notion 接单流程前自查。完整说明可以看 [Notion 管理接单流程怎么用](/blog/notion-freelance-workflow)。清单目标不是做漂亮仪表盘，而是让每个项目都有状态、下一步、交付物和复盘记录。

本文保持草稿状态，不要求你现在注册或授权 Notion，不建议保存客户敏感资料。真实共享、客户资料字段、自动化连接和团队权限都先进入明日待办。

## 适合谁

适合正在练习 AI 辅助接单、准备整理作品集或已经开始收到少量客户咨询的新手。你可以用它把线索、报价、交付和复盘放在一个地方。

也适合还没有真实客户的人。先用模拟项目练习流程，比直接把真实客户资料塞进表格更安全。

## 不适合谁

不适合想用 Notion 自动替自己联系客户、报价、承诺结果或处理客户敏感数据的人。Notion 是记录工具，不是责任转移工具。

也不适合正在管理复杂团队权限、客户合同库、财务资料或生产系统账号的人。这类内容需要专业权限设计和人工复核。

## 具体步骤

1. 建一个私人页面，不公开分享。
2. 建一个项目数据库，只放必要字段。
3. 用 3 个模拟项目测试状态流。
4. 添加低敏交付记录和复盘。
5. 检查是否存在客户隐私、权限或授权风险。
6. 把未确认的共享、自动化和模板发布事项写入明日待办。

## 1. 项目表字段检查

- [ ] 项目名称字段存在。
- [ ] 来源字段存在。
- [ ] 状态字段存在。
- [ ] 风险等级字段存在。
- [ ] 下一步字段存在。
- [ ] 截止时间字段存在。
- [ ] 交付物字段存在。
- [ ] 复盘字段存在。

不要一开始就加太多字段。第一版目标是可用，不是完整。

## 2. 状态流检查

- [ ] 线索：只是看到需求，还没判断。
- [ ] 待确认：需要继续问客户问题。
- [ ] 已报价：已经发出范围和价格。
- [ ] 进行中：正在执行低风险任务。
- [ ] 待验收：已经交付，等待客户确认。
- [ ] 已完成：交付和记录都保存好。
- [ ] 已放弃：需求不清、风险过高或不适合新手。

状态流要允许暂停和放弃。不是每个项目都应该推进。

## 3. 客户资料边界检查

- [ ] 表格只保存需求摘要，不保存客户敏感原文。
- [ ] 不保存密码、支付资料、身份证明、生产数据库或高权限链接。
- [ ] 客户姓名、邮箱、聊天记录链接是否需要保存，已经列为复核项。
- [ ] 如需保存截图，先确认截图里没有隐私信息。
- [ ] 如需共享给客户，先复制一个只含必要字段的视图。

## 4. 交付记录检查

- [ ] 每个项目都有原始需求摘要。
- [ ] 每个项目都有报价或范围说明。
- [ ] 每个项目都有交付物链接或说明。
- [ ] 每个项目都有验收方式。
- [ ] 每个项目结束后有复盘。

交付记录不需要华丽，但要能回答：客户要什么、你做了什么、怎么验证、还有什么限制。

## 风险提醒

不要把 Notion 页面随手公开。公开页面、访客权限、数据库视图和复制权限都可能带来信息暴露。

不要把自动化连接提前接上。表单、邮件、Zapier、Make、Notion API 都会改变数据流向，未复核前先不要授权。

不要把模板包装成接单结果。模板只能帮你记录流程，不能替你获得客户、完成交付或避免争议。

## 5. 明日人工复核清单

- [ ] Notion 当前分享权限、公开页面、访客权限和数据库视图设置。
- [ ] 示例项目是否完全脱敏。
- [ ] 是否需要客户同意保存需求摘要、聊天链接或交付截图。
- [ ] 是否连接表单、邮件、Zapier、Make 或 Notion API。
- [ ] 模板下载页面是否需要增加隐私提示和使用边界。
- [ ] 是否需要提供一份只读模板和一份私用模板。
