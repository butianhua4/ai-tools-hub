# 小网站交付前检查清单

小网站交付前，最怕“主观觉得完成”。一份清单能把检查变成证据：哪些页面打开过，哪些按钮点过，哪些命令跑过，哪些事项还需要客户确认。Codex 可以帮你生成初始清单，但每一项都要人工验证。

完整流程可读 [用 Codex 做小网站交付前检查](/blog/codex-small-website-delivery-check)。如果已经出现返工、错链或移动端问题，可以读 [常见错误和修复顺序](/blog/codex-small-website-delivery-check-mistakes)。

## 适合谁

适合准备交付落地页、作品集页、服务页、模板站、个人网站或小型企业站的新手。你可能已经完成页面，但还没确定客户验收前要检查哪些点。

也适合做项目时做交付说明。清单不是形式主义，它能减少客户反复问“这个测了吗”的沟通成本。

## 不适合谁

不适合大型业务系统、支付系统、登录后台、订单系统和真实用户数据平台。这些需要更完整的测试和安全审查。

也不适合把 AI 生成页面直接交付的人。AI 输出必须经过真实浏览、点击、构建和内容复核。

## 风险提醒

小网站很容易遗漏公开信息。客户名称、电话、邮箱、地址、价格、隐私政策、免责声明、版权、图片授权、备案或地区要求，都可能影响交付。

另外，部署和域名经常需要客户账号。需要登录、授权、付款、DNS 修改或邮件服务配置时，先列为客户确认，不要自行处理。

## 具体步骤

### 1. 页面路径

- 首页是否可打开。
- 导航里的每个页面是否可打开。
- 页脚链接是否可打开。
- 404 页面是否正常。
- 所有 CTA 是否指向正确位置。

如果页面较多，先列 URL 表，再逐个检查。

### 2. 移动端

- 导航是否可用。
- 标题是否换行正常。
- 按钮是否被遮挡。
- 图片是否变形。
- 是否出现横向滚动。
- 表单字段是否容易填写。

移动端问题往往比桌面端更容易被客户发现。

### 3. 表单和链接

- 邮件链接是否正确。
- 电话链接是否正确。
- 表单是否能提交。
- 下载按钮是否有效。
- 外部链接是否打开到正确页面。
- 社交链接是否为客户确认版本。

如果表单需要 API key、邮箱服务或第三方平台授权，要记录为待确认。

### 4. 内容和品牌

- 客户名称是否正确。
- logo 是否为最终版。
- 标题和副标题是否仍有模板文案。
- 图片是否有授权。
- 价格、服务范围和承诺是否谨慎。
- 是否有隐私政策或免责声明入口。

这一项需要客户最终确认，尤其是品牌和价格。

### 5. 技术检查

- lint 是否通过。
- build 是否通过。
- 关键页面是否没有控制台明显错误。
- sitemap 是否不包含草稿。
- robots 是否符合当前发布策略。
- 页面 title 和 description 是否基本准确。

遇到报错时，先用 [报错解释器](/tools/error-explainer)拆日志。

### 6. 交付记录

- 已检查页面列表。
- 已运行命令。
- 截图或录屏。
- 未覆盖范围。
- 客户待确认事项。
- 下一步建议。

可用 [模板库](/templates) 保存交付记录，用 [报价计算器](/tools/pricing-calculator)估算复核和返工时间。

## 审核结论怎么写

建议分成四类：可交付、需客户确认、需修复后交付、暂不交付。可交付表示页面和核心路径已验证；需客户确认表示内容、域名、账号或授权还没完成；需修复后交付表示发现明确 bug；暂不交付表示风险超出当前范围。

这个结论比“差不多好了”更适合项目沟通。

## 客户确认怎么留痕

交付前最好把客户需要确认的事项单独列出来，而不是夹在聊天里。常见确认项包括：正式 logo、最终联系方式、价格和服务范围、图片授权、表单收件邮箱、域名解析、部署账号、隐私政策和免责声明。只要这些没有确认，就不要把结论写成完全可交付。

如果客户临时改文案或素材，也要更新检查记录。很多返工不是代码问题，而是版本混乱：你按旧电话检查，客户后来发新电话；你按临时域名截图，客户后来要正式域名。记录每次确认的时间和版本，能减少争议。

## 交付报告示例

可以把最终结果写成简短报告：

```text
本次已检查：首页、服务页、联系页、模板页。
已验证：桌面端、移动端导航、主要 CTA、构建命令。
需客户确认：正式邮箱、图片授权、域名解析。
暂未覆盖：真实表单收件、第三方统计、长期监控。
建议下一步：客户确认账号和素材后，再做上线前复查。
```

这类报告不长，但比只发一句“检查完成”更专业，也方便后续做作品集复盘。

## CTA：下一步

把你的小网站链接按这份清单过一遍。每发现一个问题，写成“问题、截图、影响、下一步”，不要只在脑子里记着。
