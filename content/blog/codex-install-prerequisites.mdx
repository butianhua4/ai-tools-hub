# 安装 Codex 前要准备什么

很多新手一上来就搜索“怎么安装 Codex”，但真正容易卡住的地方往往不是安装命令，而是电脑环境没有准备好：终端不会用、Node 版本混乱、Git 没配置、网络访问不稳定、项目权限不清楚、报错日志不会保存。安装前先把这些基础条件检查一遍，后面排错会轻松很多。

这篇文章不是鼓励你立刻改客户项目，而是帮你建立一个可复核的本地环境准备流程。若你只想要短表版，可以看 [Codex 安装前置条件检查清单](/blog/codex-install-prerequisites-checklist)；如果已经装到一半报错，可以看 [常见错误和修复顺序](/blog/codex-install-prerequisites-mistakes)。

## 适合谁

适合刚准备使用 Codex、Claude Code、ChatGPT 或其他 AI 编程工具的新手。你可能还不熟悉 PowerShell、Terminal、VS Code、Node、Git，但希望先把环境打稳，再开始做练习项目。

也适合想做 AI 工具实践准备的人。项目不是只会打开工具就够了，你还需要能解释自己的环境、保存操作记录、复现报错、判断哪些问题需要客户或平台授权。安装前置检查能帮你少踩一点硬件和配置层面的坑。

## 不适合谁

不适合想跳过基础环境、直接在客户生产项目里试错的人。没有准备好终端、版本管理和回退方式，就不应该接触重要项目。

也不适合把安装成功等同于可以项目的人。安装只是第一步，后面还要练习读需求、运行项目、检查差异、写交付记录和识别风险。

## 风险提醒

安装类教程最容易忽略权限问题。公司电脑、客户电脑、学校电脑、云服务器和私人电脑的权限不同，能安装什么、能访问什么、能否保存凭据，都需要确认。不要为了跑通命令而随意关闭安全设置、暴露密钥或使用不明来源脚本。

另一个风险是账号和付费服务。某些工具、模型、平台或部署服务可能需要登录、授权或付费。遇到这类事项，先记录为待办或客户确认项，不要替客户注册，不要使用自己的账号混入客户项目。

## 具体步骤

1. 确认操作系统。记录你使用的是 Windows、macOS 还是 Linux，以及大致版本。遇到教程命令不同，先按系统区分。

2. 准备终端。Windows 用户至少要能打开 PowerShell；macOS 用户要能打开 Terminal。先练习 `pwd`、`ls` 或 `dir`、`cd` 这些基础命令。

3. 检查 Node。很多前端项目依赖 Node 和 npm。先运行 `node -v` 和 `npm -v`，记录版本。如果命令不存在，不要急着乱装多个版本，先查项目要求。

4. 检查 Git。运行 `git --version`，确认能查看版本。客户项目通常需要分支、提交、回退和差异检查，Git 是基础工具。

5. 准备编辑器。VS Code 或类似编辑器要能打开项目文件夹，能查看终端，能搜索文件。不要只在网页里复制粘贴代码。

6. 准备一个练习目录。不要一开始就在桌面乱放文件。建议建立一个 `ai-practice` 或类似目录，用来放练习项目、日志和截图。

7. 保存验证记录。把每一步命令输出、报错截图、安装来源和版本号记录下来。遇到报错时，可以用 [报错解释器](/tools/error-explainer)拆成现象、可能原因和下一步。

## 安装前要问自己的问题

你能不能说清当前电脑系统和终端？能不能找到项目所在文件夹？能不能运行最基础的版本命令？能不能把报错复制出来？能不能说明这个项目是自己的练习项目，还是客户项目？能不能在出错时停止，而不是继续乱改？

如果这些问题答不上来，先不要追求复杂工具。把环境准备当成第一份小练习，做完后可以整理成作品集里的“开发环境准备记录”。相关素材整理可以参考 [用 Codex 准备作品集页面素材](/blog/codex-portfolio-page-assets)。

## 项目前怎么用这套流程

接客户项目之前，你可以先把前置条件写成确认清单：客户是否提供仓库、是否有运行说明、是否需要测试账号、是否涉及私有数据、是否要求特定 Node 版本、是否允许你在本地运行。准备报价时，可以用 [报价计算器](/tools/pricing-calculator)估算环境搭建和排错时间。

如果客户需求还很散，可以用 [Proposal 生成器](/tools/proposal-generator)整理确认问题，但要人工删掉不确定承诺。需要固定表格时，可以从 [模板库](/templates) 找环境检查或交付记录模板。

## CTA：下一步

先不要安装一堆工具。打开终端，记录系统、Node、npm、Git 和项目目录这五项信息。完成后，再用 [安装前置条件检查清单](/blog/codex-install-prerequisites-checklist) 逐项确认。
