# Codex 怎么做第一个落地页：常见错误和解决步骤

用 Codex 做第一个落地页，最容易踩的坑不是“代码不会写”，而是范围失控。新手常常一开始只想做一个简单页面，结果让 AI 加了表单、支付、动画、后台、数据库和一堆自己看不懂的组件。页面看起来有东西了，但无法验证，也无法交付。

这篇文章整理常见错误和纠正步骤。它适合做练习复盘，不代表读完就能接复杂落地页项目。AI 可以辅助生成页面，但最终判断、测试、文案审核和平台合规都必须由人负责。

## 适合谁

适合已经开始用 Codex 做页面，但经常遇到“改完不知道对不对”“手机端乱了”“构建失败”“页面很漂亮但说不清用途”的新手。也适合准备做作品集练习的人，用来检查自己的第一个落地页是否过度复杂。

## 不适合谁

不适合想复制别人页面、伪造案例、批量生成相似页面或直接拿未经验证的 AI 输出交给客户的人。也不适合正在处理支付、登录、生产数据库或真实广告投放页面的新手；这些项目需要更高的工程和业务判断。

## 错误一：主题太大

错误表现：你让 Codex “做一个 AI 自动化项目平台页面”“做一个完整 SaaS 首页”“做一个能卖课的转化页”。这些需求太宽，Codex 只能猜。它可能生成很多模块，但每个模块都不够真实。

纠正步骤：把主题缩成一个小页面。例如“AI 工具学习每日执行表下载页”“客户需求沟通表介绍页”“个人作品集入口页”。一个页面只解决一个问题，只保留一个主要 CTA。

## 错误二：素材没准备好

错误表现：没有标题、目标用户、功能点、CTA、图片来源和风险说明，就直接让 Codex 写页面。最后页面里会出现空泛表达，甚至带来不真实的承诺。

纠正步骤：先写素材清单，再让 Codex 设计结构。最少准备 H1、副标题、适合谁、不适合谁、3 个核心点、CTA 和免责声明。没有授权图片就先不用图片，宁可简洁，也不要抓取别人素材。

## 错误三：一次改太多

错误表现：你让 Codex 同时改页面结构、样式、文案、表单、动画和部署配置。改完以后页面出错，你不知道是哪一步造成的。

纠正步骤：每次只改一个范围。先让它只输出结构；确认后只改页面文件；再单独处理样式；最后再检查移动端。每一步都看 `git status`，知道文件变化。

## 错误四：忽视移动端

错误表现：桌面端看起来不错，手机上标题太长、按钮挤在一起、卡片堆叠混乱、CTA 看不到。落地页如果手机端不清楚，练习价值会大打折扣。

纠正步骤：至少检查 375px、768px 和桌面宽度。看 H1 是否过长、按钮是否换行、卡片间距是否正常、长中文是否溢出。保存截图，作为作品集记录的一部分。

## 错误五：跳过构建

错误表现：浏览器预览勉强能看，但 `npm run build` 失败。新手可能以为页面完成了，实际上它无法稳定部署。

纠正步骤：每次完成一个阶段都运行：

```bash
git status
npm run build
```

如果构建失败，不要随便删除文件、关闭检查或忽略错误。把错误复制到 [报错解释器](/tools/error-explainer)，先理解大概原因。

## 错误六：把练习页当客户案例

错误表现：页面是自己练习做的，却包装成客户项目，甚至写成已经帮助别人获得具体业务结果。这会损害长期信誉。

纠正步骤：作品集可以写“个人练习项目”或“AI 辅助页面练习”。说明目标、工具、过程和验证结果，不要伪装成真实商业案例。真实比夸张更适合新手。

## 具体步骤

1. 先写一句话主题，确认页面只解决一个问题。
2. 准备标题、副标题、适合谁、不适合谁、CTA 和风险说明。
3. 让 Codex 先检查结构，不要直接改代码。
4. 小范围修改一个页面或组件。
5. 运行 `git status` 和 `npm run build`。
6. 检查移动端、平板和桌面效果。
7. 保存截图、提示词、改动文件和复盘。

## 可以复制的修正提示词

```text
这个落地页现在范围太大。请先不要继续加功能。
帮我把页面缩小成一个新手练习版本：
1. 只保留一个主题。
2. 只保留一个主要 CTA。
3. 不接支付、登录、后台、数据库。
4. 加上适合谁、不适合谁和风险提醒。
5. 告诉我应该如何验证移动端和构建结果。
```

## 风险提醒

不要复制别人落地页的整段文案、图片和案例，不要伪造客户项目，不要承诺具体收入、转化率或广告效果。不要为了让页面看起来完整而接入你不理解的支付、追踪脚本或生产账号。

如果以后要把练习页用于项目展示，请明确标注它是个人练习或演示项目。真实、可解释、可验证，比夸张包装更适合长期积累。

## 总结

第一个落地页最怕范围失控。新手应该从小主题开始，准备真实素材，让 Codex 小步修改，然后用构建结果和截图验证。不要急着把练习包装成商业案例，也不要碰自己无法解释的复杂功能。

你真正要练的不是“生成页面”，而是“控制范围、验证结果、记录过程”。这才是以后做小项目的基础。
