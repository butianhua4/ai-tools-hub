# 每天项目练习记录检查清单

这份清单用于把每天的 AI 项目练习写成可复查记录。完整方法见 [新手如何记录每天的项目练习](/blog/daily-ai-freelance-practice-log)。每天不需要写长文章，但要留下足够证据，证明你做了什么、怎么验证、哪些还不能承诺。

## 适合谁

适合每天练习一点 AI 工具实践技能的新手。你可能在练网页修改、报错排查、客户回复、报价估算、模板整理或工具对比。

也适合准备做作品集的人。作品集不是等到大项目出现才开始写，而是从每天小任务的真实记录里筛选出来。

## 不适合谁

不适合只想写漂亮总结、但没有真实操作记录的人。没有原始问题、没有过程、没有验证截图的记录，很难让客户相信。

也不适合把客户隐私和平台敏感信息直接写进练习日志的人。练习记录应该保护真实姓名、邮箱、订单号、仓库地址、付款信息和未授权截图。

## 风险提醒

清单不是为了制造“我每天都很强”的感觉，而是为了发现能力边界。写出没完成的部分、需要复核的部分、需要客户授权的部分，反而更有价值。

如果当天练习涉及账号、仓库、付款、客户资料或平台规则，把它标成敏感练习，不要公开发布。公开作品集时只保留脱敏后的问题、过程和结果。

## 具体步骤

### 1. 任务目标

- 今天练什么：
- 为什么选这个：
- 预计用时：
- 完成标准：

完成标准要具体。比如“让页面按钮颜色变化并通过 build”，比“练习改网页”更清楚。

### 2. 原始材料

- 原始报错：
- 原始需求：
- 截图或链接：
- 已脱敏的信息：

如果不能公开原始材料，就写“已脱敏记录”。不要为了做作品集暴露客户或平台信息。

### 3. AI 使用过程

- 使用了哪个工具：
- 输入了哪些上下文：
- AI 给了哪些建议：
- 采纳了哪些：
- 拒绝了哪些：

这一项很关键。客户想知道你会不会判断 AI 输出，而不是只会复制。

### 4. 人工判断

- 哪一步是你自己判断的：
- 哪一步需要复核：
- 哪一步不能承诺：
- 是否涉及客户授权：

如果有授权事项，把它写成待办，而不是假装已经解决。

### 5. 验证结果

- 运行了什么命令：
- 看了哪个页面：
- 结果截图在哪里：
- 是否还有失败项：

代码类练习建议至少记录一个命令结果。文案类练习建议保留前后对比。

### 6. 客户表达

练习结束后写一句客户能听懂的话：

```text
本次练习定位到问题来自 Git 认证配置，代码本身还没有进入验证阶段；下一步需要确认仓库权限。
```

这句话训练的是交付沟通，不只是技术复盘。

## 每周复盘字段

每天记录之外，每周还要做一次筛选：哪三条记录可以变成作品集，哪三条暴露了能力缺口，哪一类任务暂时不适合接，哪一个模板可以复用。

一周复盘不要只统计数量。更有用的是看趋势：你是不是越来越会写确认问题、越来越会保留验证证据、越来越能把风险讲清楚。

## 明日待办字段

每天结束前，单独写一栏明日待办。待办不要写成模糊的“继续学习”，而要写成可以执行的小动作，比如复现一个报错、补一张截图、把客户回复改成英文、或者确认某个工具是否需要登录。

如果待办涉及注册、授权、客户资料、私有仓库或平台账号，要标记负责人。能自己做的写“我明天验证”；需要客户做的写“等待客户授权”；需要人工复核的写“明天请人检查”。这样第二天打开记录时，不会把外部等待误认为自己拖延。

明日待办也能保护项目边界。比如今天练习到 Git 认证失败，发现需要仓库所有者添加协作者，就不要写“明天继续修好”，而要写“明日等待客户添加协作者后再验证 fetch”。这种记录更准确，也更安全。
