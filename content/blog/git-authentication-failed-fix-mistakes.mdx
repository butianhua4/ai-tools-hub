# git authentication failed 怎么解决？密码认证已停，用 PAT 或 SSH

git push / clone 报 `remote: Support for password authentication was removed`、`Authentication failed`、或 `could not read Username for 'https://github.com'`，根本原因是同一个：**GitHub 早已停用账号密码认证**。HTTPS 现在要用 Personal Access Token（PAT），或者干脆改用 SSH。

先看你的远程地址是哪种，两条路修法不同：
```bash
git remote -v
# https://github.com/... → 走 HTTPS + PAT（见一）
# git@github.com:...     → 走 SSH 密钥（见二）
```

## 一、HTTPS：用 Personal Access Token 当密码

**成因**：HTTPS 方式下，git 提示输入密码，但 GitHub 不再接受密码，要的是 PAT。

**步骤**：
1. GitHub → Settings → Developer settings → Personal access tokens → 生成一个（勾上 `repo` 权限）。
2. push 时用户名填 GitHub 用户名，**密码处粘贴这个 token**。
3. 想免每次输入，用凭据管理器缓存：
```bash
git config --global credential.helper store   # 或 macOS 用 osxkeychain
```

**注意**：token 等于密码，别提交进代码、别公开；过期了要重新生成。

## 二、SSH：配密钥（一次配好，长期免密）

**成因**：用 `git@github.com:` 地址但本机没配 SSH key，报 `Permission denied (publickey)`。

**步骤**：
```bash
ssh-keygen -t ed25519 -C "你的邮箱"     # 一路回车
cat ~/.ssh/id_ed25519.pub               # 复制这串公钥
# 粘到 GitHub → Settings → SSH and GPG keys → New SSH key
ssh -T git@github.com                    # 测试,出现 successfully authenticated 即成功
```

## 三、旧凭据没清，一直用错的在认证

**成因**：系统凭据管理器里存了旧的（错误或过期的）GitHub 账号/token，git 一直拿它去认证。

**修复**：
- Windows：控制面板 → 凭据管理器 → Windows 凭据，删掉 github 相关条目。
- macOS：钥匙串访问搜 `github.com` 删掉旧条目。
- 通用：`git config --global --unset credential.helper` 后重新认证。

## 四、HTTPS 和 SSH 之间切换

如果 PAT 太麻烦，直接把远程切到 SSH：
```bash
git remote set-url origin git@github.com:用户名/仓库名.git
```
反过来想用 HTTPS 就换成 `https://github.com/...`。

## 风险提醒

- PAT、SSH 私钥等同密码：别写进代码、别截图公开、别发给别人。
- 公司/客户私有仓库的访问权限，先确认授权再配；离职或换设备记得吊销旧 token。
