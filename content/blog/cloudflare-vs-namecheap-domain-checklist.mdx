# Cloudflare 和 Namecheap 域名选择检查清单

选择域名注册商前，先不要打开购物车。真正要确认的是：这个域名归谁、DNS 由谁管、是否要邮箱、是否要迁移、是否允许使用 Cloudflare nameserver、续费提醒谁负责、客户是否能接管账号。

完整判断可以看 [Cloudflare 和 Namecheap 域名怎么选](/blog/cloudflare-vs-namecheap-domain)。这篇是可复制的检查清单，适合你在作品集、客户小网站、Vercel 部署或静态站上线前逐项核对。

## 适合谁

适合还没有稳定域名流程的新手。你可能已经能部署网站，但每次遇到域名就开始混淆注册商、DNS、SSL、邮箱和部署平台。这份清单的目标不是替你做购买决定，而是让你知道哪些信息必须确认。

也适合准备接小型网站上线任务的人。客户让你“顺便配一下域名”时，往往比改页面更容易出事故。清单能帮你把客户材料、账号边界和交付记录放在前面。

## 不适合谁

不适合正在处理客户正式业务主域名、企业邮箱、支付回调、登录系统或生产 API 的新手独立操作。这类域名变更需要更严格的变更窗口、备份、回滚方案和客户授权。

也不适合把注册商对比当成唯一答案的人。Cloudflare 和 Namecheap 都只是选项，真正要看的是具体域名、具体 TLD、具体 DNS 需求和客户接管能力。

## 风险提醒

域名任务最怕“账号归属不清”。如果用你的账号买客户域名，后续续费、发票、授权、转移、争议都会变复杂。客户项目应尽量让客户自己持有注册商账号，你只做清单、建议、远程指导或经授权的短期协助。

涉及注册、付款、邮箱验证、迁移授权码、账号授权、企业 DNS、生产站点切换的事项，先写入待确认清单。没有确认前，不要动客户主域名。

## 具体步骤

### 1. 用途检查

- 域名是作品集、测试站还是正式官网。
- 是否有邮箱、登录、支付、API 或客户后台。
- 是否只是临时演示，还是需要长期续费。
- 是否有品牌、商标或命名风险需要客户自己判断。

用途不清时，先不要购买。

### 2. 账号检查

- 注册商账号由谁持有。
- 注册邮箱是否属于客户。
- 付款方式是谁的。
- 续费提醒发给谁。
- 是否需要发票或公司信息。

客户项目里，账号归属比工具选择更重要。

### 3. DNS 检查

- 是否只需要 A、CNAME、TXT 记录。
- 是否需要 MX、SPF、DKIM、DMARC 等邮箱记录。
- 是否需要 DNSSEC。
- 是否需要子域名委派。
- 是否已有旧记录需要复制。

DNS 记录没有备份时，不要急着切换 nameserver。

### 4. Cloudflare 检查

- 是否愿意长期使用 Cloudflare nameserver。
- 是否确认目标 TLD 支持注册或转入。
- 账号邮箱是否已验证。
- 是否理解 Cloudflare Registrar 的 nameserver 限制。
- 是否需要 Cloudflare 的 DNS、安全和缓存能力。

如果客户想随时换到别的 nameserver，这一点要提前写清。

### 5. Namecheap 检查

- 是否已有 Namecheap 账号。
- 是否使用 BasicDNS、PremiumDNS 或自定义 DNS。
- 是否需要额外购买 PremiumDNS。
- 是否知道 DNS 记录在哪个后台维护。
- 是否确认 PremiumDNS 到期后的影响。

如果客户只是想要通用注册商，Namecheap 更容易被理解，但仍要看具体 DNS 需求。

### 6. 迁移检查

- 是否需要授权码。
- 是否有注册锁或转移冷却期。
- 是否要提前复制 DNS 记录。
- 是否有邮箱和业务系统正在使用域名。
- 是否安排了验证和回滚时间。

迁移不是新手练手的第一选择。可以先写方案，再让客户或有经验的人复核。

## 最低交付记录

一次域名选择咨询，至少交付六项：推荐选项、原因、成本注意点、DNS 注意点、客户需确认事项、下一步操作。不要只写“建议用某某”，那样客户无法复查，也无法判断你有没有看见风险。

如果是接单任务，可以把交付拆成两档：第一档只做方案和清单；第二档在客户账号中协助配置 DNS。第二档需要明确授权、操作窗口和回滚方式。
