# 不会写代码怎么交付一个小网站

不会完整写代码，并不等于完全不能练习小网站交付。但你要把范围控制得很小：单页展示、简单作品集、活动页、工具导航、价格表、FAQ 页面，这类任务才适合新手用 AI 和模板练习。

真正的重点不是“AI 能不能生成网页”，而是你能不能确认需求、选择合适模板、改对内容、验证页面、说明限制，并把客户授权事项放在正确位置。小网站交付不是炫技，是把一个明确范围做完整。

## 适合谁

适合已经会使用 ChatGPT、Codex、Cursor、v0 或现成模板，但还不能独立从零写完整项目的新手。你愿意按步骤验证，而不是把 AI 生成内容直接发给客户。

也适合准备做作品集的人。你可以先用自己的练习项目，做一个真实可访问的小页面，再记录从需求到交付的全过程。

## 不适合谁

不适合接复杂后台、支付系统、用户登录、数据库、生产迁移、安全修复这类任务的人。这些任务对权限、测试和维护要求更高，不适合作为不会写代码的新手练手单。

也不适合承诺“什么网站都能做”的人。小网站交付的前提是范围清楚、内容简单、风险可控、验收标准明确。

## 风险提醒

不要让客户把域名账号、支付账号、主机密码、生产环境密钥直接发给你。需要上线时，优先让客户在平台侧添加协作者、授权项目，或由客户自己完成敏感账号操作。

另外，模板、图片、图标、字体都要注意授权。练习项目可以使用免费资源，但真实交付前要确认素材来源和使用范围。

## 具体步骤

### 1. 先确认网站范围

最小范围可以这样写：

```text
一个单页网站，包含顶部导航、简介、服务说明、3 个案例、FAQ 和联系按钮。
不包含登录、付款、数据库、后台管理和复杂动画。
```

范围越清楚，越容易交付。不要让“小网站”变成无限修改。

### 2. 准备客户材料

至少需要：品牌名、页面文案、联系方式、图片或 logo、参考网站、颜色偏好、上线方式、验收标准。没有文案时，可以先做占位稿，但要明确标记需要客户确认。

如果客户还没有域名或部署账号，把它写成待办，不要假装已经能上线。

### 3. 选择模板或工具

新手可以从模板开始，而不是从空白项目开始。模板要简单、结构清楚、容易修改。不要选那种动画很多、依赖复杂、看起来漂亮但很难维护的项目。

AI 可以帮你生成组件、改文案、解释报错，但你要自己检查页面是否能运行，内容是否真实，按钮是否指向正确位置。

### 4. 本地验证

交付前至少检查：

```bash
npm run build
```

如果是纯静态 HTML，也要在浏览器里打开检查移动端、按钮、链接、图片和文案。验证结果要写进交付记录。

### 5. 上线和授权

如果需要部署到 Vercel、Netlify、GitHub Pages 或客户自己的主机，先确认谁拥有账号、谁授权项目、谁负责域名解析。涉及注册、绑定域名、付款套餐或组织权限的事项，应由客户确认。

你可以交付代码、部署说明、预览链接或 PR，不一定非要代客户掌管所有账号。

## 交付说明怎么写

交付说明至少包含：完成了哪些页面、使用了什么模板或工具、哪些内容需要客户确认、运行或预览方式、已知限制、后续维护建议。

示例：

```text
本次完成单页展示网站的静态版本，包含首页、服务说明、案例区和联系按钮。当前不包含后台、登录、付款和表单存储。上线前需要客户确认最终文案、图片授权和域名解析方式。
```

这类说明能降低误会，也能保护你的交付边界。
