# Upwork 任务风险检查清单

筛选 Upwork 任务时，新手最容易问“我能不能做”，但更稳的问题是“这个任务材料够不够、范围清不清、权限是否安全、验收是否明确、平台规则是否合规”。下面这份清单适合在写 proposal 前使用，也适合让 Codex 帮你做第一轮拆解。

完整流程可读 [用 Codex 判断 Upwork 任务风险](/blog/codex-upwork-job-risk-check)。如果已经遇到范围失控或客户要求不清，可以看 [常见错误和修复顺序](/blog/codex-upwork-job-risk-check-mistakes)。

## 适合谁

适合刚开始筛选 Upwork 任务的新手。你可能会修小 bug、整理 README、检查移动端或做小网站，但还不知道哪些任务适合练手。

也适合已经写 proposal 但回复率低的人。很多时候问题不是文案不够漂亮，而是任务本身不适合，或者你没有先问关键问题。

## 不适合谁

不适合批量复制 proposal 的人。没有读懂任务就投，容易接到范围不清、预算不匹配或风险过高的工作。

也不适合试图绕开平台流程的人。付款、沟通、合同和交付都应遵守平台规则。

## 风险提醒

清单不能替你保证任务安全。客户背景、平台规则、项目技术、权限要求都会变化。遇到敏感权限、生产数据、第三方账号、支付、登录和安全问题时，要提高风险等级。

任何需要客户注册、登录、购买、授权或提供凭据的事项，都应写入待确认列表。不要把自己的账号和客户项目混在一起。

## 具体步骤

### 1. 任务信息

- 标题是否具体。
- 描述是否说明目标。
- 是否有截图、链接或仓库。
- 截止时间是否合理。
- 预算是否和范围匹配。

只有一句“easy task”的项目，不能直接当作简单任务。

### 2. 范围

- 是否只修一个明确问题。
- 是否包含设计、开发、部署、文案、SEO、维护等多项。
- 是否说明交付物。
- 是否有验收标准。
- 是否存在“顺便”很多的表达。

范围越模糊，越适合先问，不适合直接报价。

### 3. 权限

- 是否需要私有仓库。
- 是否需要服务器。
- 是否需要域名或 DNS。
- 是否需要数据库。
- 是否需要 API key、token、后台或客户账号。

权限越多，风险越高。新手应优先选择权限少、可本地验证的小任务。

### 4. 技术匹配

- 技术栈是否熟悉。
- 是否能在本地复现。
- 是否有 README 或运行说明。
- 是否涉及老项目依赖。
- 是否有测试或构建方式。

不熟悉不一定不能投，但要把学习和排查时间算进报价。

### 5. 客户沟通

- 客户是否愿意回答问题。
- 是否要求立即开始。
- 是否拒绝提供必要材料。
- 是否把结果说得过满。
- 是否要求不合理的免费试做。

沟通信号往往比技术栈更早暴露风险。

### 6. 结论

- 可投。
- 先问。
- 跳过。
- 只做诊断。

把结论写下来，再决定是否用 [Proposal 生成器](/tools/proposal-generator)起草回复。报价可以用 [报价计算器](/tools/pricing-calculator)做时间估算。

## 任务评分方式

可以给每个任务打 0 到 2 分：范围清楚 2 分，材料齐全 2 分，权限低风险 2 分，技术熟悉 2 分，验收明确 2 分。低于 6 分就先问或跳过，不要急着投。

这个评分不是绝对标准，但能防止情绪化项目。看到高预算时也要打分，别让预算遮住风险。

## 确认问题怎么写

确认问题要短而具体。不要一次发十几个问题吓到客户，也不要只问“can you share more details”。更稳的写法是围绕三个点：目标结果、现有材料、验收方式。比如可以问当前链接或仓库在哪里、什么结果算完成、是否有测试环境。

如果任务涉及权限，还要问最小权限，而不是直接索要全部账号。比如可以问客户是否有测试环境、是否能提供只读仓库、是否能由客户自己配置密钥。这样既保护客户，也保护你。

## 投标前最后复核

发送 proposal 前，再看一次任务是否仍然适合自己。新手常见状态是：写着写着就把不确定内容写成了承诺。检查是否出现“我可以完整解决所有问题”“我可以立即上线”“我熟悉全部技术栈”这类过满表达。没有验证过的内容，要改成“我会先检查”“我需要确认”“我可以从诊断开始”。

如果你准备用 [Proposal 生成器](/tools/proposal-generator)起草，也要人工删改。AI 生成的英文通常很顺，但可能把边界写得太满。项目早期，谨慎比漂亮更重要。

## 记录模板

```text
任务链接：
客户目标：
缺失材料：
权限要求：
风险等级：
确认问题：
项目结论：
```

可以把这份模板保存到 [模板库](/templates) 的项目筛选记录里，持续积累自己的判断案例。

选 5 个 Upwork 任务，不写 proposal，只按这份清单打分。练到能快速说出“为什么不投”，你的项目判断会稳很多。
