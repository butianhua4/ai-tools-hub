# Codex 安装失败怎么办：新手排查清单

Codex 安装失败时，先不要把问题归因给工具本身。安装失败通常来自环境、权限、网络、包管理器、路径或版本差异。新手要做的第一步，是把“我装不上”改写成一份可以复查的安装记录。

这篇仍是草稿。发布前需要人工核对官方安装文档、真实命令输出和不同系统差异，不能把本文当成最终安装指南。

## 适合谁

适合刚开始使用 Codex、在本地安装或运行命令时失败的新手。你可能还不熟悉 Node.js、npm、PowerShell、Terminal 或 PATH，但愿意按顺序保存日志并逐项排除。

也适合准备帮客户处理安装报错的人。你可以先用这份清单判断客户缺什么材料，再决定是否进入诊断或报价。

## 不适合谁

不适合正在处理公司受管设备、生产服务器、客户云账号或需要管理员授权的环境。安装问题看起来小，但如果牵涉组织权限、安全策略、代理网络或敏感账号，新手不应该独立修改系统配置。

也不适合想复制一条命令就交付的人。安装失败需要看具体日志，不能用固定答案覆盖所有环境。

## 先收集哪些信息

安装问题最怕信息不完整。你至少要记录：

1. 操作系统和终端类型，例如 Windows PowerShell、macOS Terminal 或 Linux shell。
2. 完整安装命令，不要只写“安装失败”。
3. 完整报错输出，包括命令上方和下方的相关日志。
4. 当前目录、项目路径和是否在公司/学校网络下。
5. Node.js、npm 或其他包管理器版本。
6. 是否使用代理、镜像源、VPN、受管电脑或管理员账户。

有了这些材料，AI 才能更可靠地帮你解释报错；没有这些材料，只是在猜。

## 具体步骤

1. 先保存完整日志，截图和文本都保留一份。
2. 确认当前终端是否能识别基础命令，例如 `node -v`、`npm -v`、`git --version`。
3. 检查安装命令是否来自官方或项目文档，发布前需要人工核对最新说明。
4. 判断错误类型：网络下载失败、权限不足、路径异常、版本不匹配，还是命令拼写错误。
5. 如果是权限问题，先确认当前目录和用户身份，不要直接扩大系统权限。
6. 如果是网络或代理问题，记录网络环境，让客户或自己确认是否需要平台授权或网络设置。
7. 每次只改一个变量，改完重新运行同一条命令，并保存前后结果。
8. 仍然失败时，把日志整理成最小问题描述，再交给人工复核。

```text
是否使用代理或镜像源：

```

可以先运行这些只读命令：

```bash
node -v
npm -v
git --version
pwd
```

Windows PowerShell 里可以补充：

```powershell
Get-Location
Get-Command node
Get-Command npm
```

## 接单时怎么问客户

如果客户说 “Codex install failed”，不要立刻承诺修好。先要求客户提供系统、完整日志、安装命令、是否可以远程看仓库或复现环境、是否允许创建测试记录。

如果需要客户注册、登录、添加授权或调整公司设备策略，把它写成明日待办或客户侧操作，不要把敏感凭据放进聊天记录。

## 一个新手可处理的例子

比较适合新手处理的情况，是客户能提供完整终端日志，报错只发生在本地项目安装阶段，且不需要访问生产服务器。比如客户已经有 GitHub 仓库，你只需要在自己的测试环境里复现安装失败，记录 Node.js 和 npm 版本，确认是不是依赖版本、路径或包管理器命令不一致。

这类任务的交付物可以很清楚：一份诊断记录、一组建议命令、一段客户可以自己执行的修复步骤。如果需要改仓库文件，也应该通过测试分支或 PR 交付，并说明你验证过哪些命令。

不适合新手直接处理的情况，是客户说公司电脑被策略限制、网络代理需要管理员配置、安装过程要求登录内部系统，或者必须使用客户的远程桌面和账号。遇到这类情况，最好把问题拆成“客户侧授权”和“技术侧诊断”两部分。授权动作由客户或管理员完成，你只记录需要什么信息。

## 交付前怎么说明

交付时不要只写“安装好了”。更好的说明是：原始报错是什么、判断原因是什么、你改了什么或建议客户改什么、验证命令是什么、还有哪些环境差异需要客户自己确认。

如果你没有在客户真实环境里验证，就不要写成已经完全解决。可以写“本地复现环境已通过，客户设备仍需按清单确认网络和权限”。这种表达虽然不花哨，但对接单信誉更稳。

## 风险提醒

不要让客户发送密码、私钥、token、公司后台截图或不必要的管理员账号。不要在不理解影响范围的情况下修改系统 PATH、shell 配置、全局 npm 权限或安全策略。

安装类问题可以做成诊断单，但如果牵涉公司设备、网络代理、组织权限或生产服务器，应当升级为人工复核。
