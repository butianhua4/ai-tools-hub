# Payoneer Wise PayPal 怎么选择：新手检查清单

这份清单用于比较 Payoneer、Wise、PayPal 前的自查。完整解释可以先看 [Payoneer Wise PayPal 怎么选择](/blog/payoneer-wise-paypal-choose)。清单的目标不是让你马上注册，而是帮你判断自己下一步该查什么、记录什么、暂停什么。

本文保持草稿状态，不替你提交真实身份资料、不替你绑定银行账户、不替你判断税务或法律责任。只要涉及注册、实名、提现、大额交易、争议处理或费用确认，都应进入明日待办。

## 适合谁

适合准备接海外小单、客户直接付款、模板下载或平台提现的新手。你可能还没有真实收入，但希望先知道自己是否需要 Payoneer、Wise、PayPal。

也适合已经有一个客户咨询的人。你可以用清单把客户地区、付款币种、交付内容、平台规则和记录保存先整理出来，再决定是否需要进一步查证。

## 不适合谁

不适合正在处理公司财务、税务申报、大额交易、敏感账户、客户生产支付系统或复杂合规问题的人。这类事项需要专业复核。

也不适合想用一个工具解决所有接单问题的人。收款方式只是流程的一环，前面还要有清楚的需求、报价、验收、交付和证据记录。

## 具体步骤

1. 先完成收入来源检查。
2. 再完成客户与平台检查。
3. 分别标记 Payoneer、Wise、PayPal 的可确认项和不可确认项。
4. 把不可确认项写进明日人工复核清单。
5. 用记录模板保存判断过程。
6. 在没有完成官方文档查证前，不注册不必要账户，不上传敏感资料。

## 1. 收入来源检查

- [ ] 我的收入来自自由职业平台。
- [ ] 我的收入来自客户直接付款。
- [ ] 我的收入来自模板、数字产品或下载页面。
- [ ] 我的收入来自订阅、软件或独立站。
- [ ] 我还没有真实收入，只是在提前准备。

如果还没有真实收入，优先级通常不是马上注册工具，而是完善作品集、联系表单、报价说明和交付模板。

## 2. 客户与平台检查

- [ ] 我知道客户在哪个国家或地区。
- [ ] 我知道客户想用什么币种付款。
- [ ] 我知道订单金额大概是多少。
- [ ] 我知道客户是否来自一个已有平台。
- [ ] 我已经查看平台是否指定付款或提现方式。
- [ ] 我知道付款前需要保存哪些聊天、报价和交付证据。

如果客户来自平台，先看平台规则。如果客户直接付款，先看合同、发票、退款、交付和争议记录能不能说清楚。

## 3. Payoneer 检查

- [ ] 我确认目标平台是否支持 Payoneer。
- [ ] 我确认自己所在地区是否能开通或使用相关服务。
- [ ] 我确认提现到本地银行需要哪些资料。
- [ ] 我确认费用、到账时间和账户限制需要明日查证。
- [ ] 我没有因为别人推荐就立即上传身份或银行资料。

## 4. Wise 检查

- [ ] 我确认客户是否能按 Wise 提供的收款信息付款。
- [ ] 我确认自己所在地区是否支持需要的账户能力。
- [ ] 我确认币种、转账费用、汇率和到账记录需要明日查证。
- [ ] 我准备保存转账说明、订单编号和客户确认。
- [ ] 我没有把 Wise 当作所有平台项目的默认收款方式。

## 5. PayPal 检查

- [ ] 我确认客户是否愿意使用 PayPal。
- [ ] 我确认账户类型、收款限制、费用和提现路径需要明日查证。
- [ ] 我确认争议处理、退款、付款类型和账户限制需要人工复核。
- [ ] 我准备在付款前写清服务范围和交付标准。
- [ ] 我没有把 PayPal 链接当作替代合同或交付说明的东西。

## 风险提醒

不要只看“哪个更便宜”。跨境收款还要看客户能不能付、你能不能提、记录是否完整、争议如何处理、税务资料如何保存。

不要在没有完成查证前上传敏感资料。真实账户注册、身份验证、银行绑定、税务表单和大额交易，都应该等明日人工核对后再做。

不要承诺收款一定顺利。平台审核、客户付款失败、账户限制、汇率变化、退款和争议都可能发生。新手能做的是提前记录边界和证据。

## 6. 明日人工复核清单

- [ ] Payoneer 当前地区支持、费用、提现路径、平台支持和账户资料要求。
- [ ] Wise 当前地区支持、多币种账户能力、转账费用、收款信息和限制。
- [ ] PayPal 当前账户类型、收款限制、费用、提现、退款和争议规则。
- [ ] 目标接单平台是否允许对应提现方式。
- [ ] 是否需要税务资料、发票、合同、消费者提示或额外合规说明。
- [ ] 是否有更适合当前订单的平台内置收款方式。
