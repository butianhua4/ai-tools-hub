# 新手练习项目记录表怎么用：新手检查清单

练习项目记录表写完后，还要检查它能不能被复盘。一个好记录不是流水账，而是能回答：为什么做、怎么做、怎么验证、哪里失败、下一步是什么。

这篇仍是草稿。发布前需要人工补充真实记录样例，确认没有泄漏客户信息、账号、密钥或未授权素材。

## 适合谁

适合已经开始做练习项目，并想把记录整理成作品集素材的新手。你可能正在练网页、报错、CSS、部署、Proposal 或报价，都可以用这份清单检查记录质量。

也适合每天练习但感觉没有积累的人。记录表能把碎片练习变成可复盘材料。

## 不适合谁

不适合只想快速写一堆假案例的人。也不适合把课程作业、客户资料或公司项目直接公开的人。

如果练习涉及真实客户或私有资料，要先做脱敏和授权确认，不能直接变成公开案例。

## 检查清单

逐项确认：

1. 目标是否只有一个，不是同时练十件事。
2. 输入材料是否完整，包括原始需求、截图、报错或文件。
3. 是否记录了使用的工具和命令。
4. 是否写清楚实际改动，而不是只写“修好了”。
5. 是否有验证证据，例如命令输出、截图或手动检查路径。
6. 是否记录失败和卡点。
7. 是否标出需要人工复核的内容。
8. 是否遮盖敏感信息。
9. 是否列出明日待办。
10. 是否能被改写成一段作品集案例。

如果只满足前四项，这更像操作笔记；满足八项以上，才更接近可复盘记录。

## 具体步骤

1. 先读一遍记录，删掉和本次练习无关的闲聊内容。
2. 给每个结论补证据，例如截图、命令、链接或文件名。
3. 把 AI 建议和你实际执行的动作分开写。
4. 检查是否有客户信息、账号、token、邮箱、订单号或后台截图。
5. 给失败项写下一步，而不是只写“没成功”。
6. 最后写一句总结：这次练习证明了什么，还不能证明什么。

## 可复制复核模板

```text
练习记录复核清单

目标明确：是 / 否
输入完整：是 / 否
过程可追踪：是 / 否
验证有证据：是 / 否
截图已脱敏：是 / 否
失败已记录：是 / 否
人工复核项：有 / 无
明日待办：有 / 无
是否适合改成作品集：是 / 否
```

这份复核模板可以贴在每条练习记录末尾。长期看，它比单纯堆练习数量更有用。

## 如何判断能不能公开

能公开的记录通常满足三个条件：内容是你自己的练习或已有授权；截图不含敏感信息；结果可以被验证。任何一项不满足，都先保持内部草稿。

如果你只是照着教程完成了一遍，也可以记录，但不要写成独立商业项目。可以写“跟随教程复现，并补充了自己的排查记录”。

## 复盘时问自己的问题

每条记录最后可以问自己五个问题：我能否用一句话说明练习目标；我能否指出最关键的一处改动；我是否保存了验证证据；我是否知道还有哪里没验证；这条记录是否适合公开。如果其中两个以上答不上来，就先继续补记录。

新手不要害怕记录“不懂”。真正有用的复盘不是把自己写得很顺，而是把不确定的地方标出来。比如“我还不确定为什么这个依赖冲突发生”“我只在本地验证，未做预览部署”，这些信息能帮助你下一次更快定位问题。

当你连续积累 5 条合格记录，再挑其中 1 条改成作品集案例，会比一次性编 5 个案例更稳。

## 风险提醒

不要公开客户隐私、私有仓库、密钥、后台截图、交易信息和聊天记录。不要把没有验证的 AI 输出写成项目结果。不要承诺收入、成交或排名效果。

需要注册平台、连接工具、申请客户授权或发布作品集的事项，放进明日待办。

## 明日待办

- 挑 3 条练习记录做脱敏复核。
- 补充命令输出或截图证据。
- 人工确认哪些记录可以公开。
- 把一条合格记录改写成作品集草稿。
