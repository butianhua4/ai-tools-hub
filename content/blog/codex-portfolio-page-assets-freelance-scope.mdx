# 作品集素材整理项目能不能接

作品集素材整理是一类很适合新手观察的服务：客户有项目经历、截图或零散记录，但不知道怎么变成案例页面。你可以用 Codex 帮忙梳理结构、提炼要点、生成草稿，但项目前必须判断授权、素材完整度和交付边界。否则很容易从“整理素材”变成“设计页面、写文案、修图、做网站、处理隐私和承担效果承诺”的混合任务。

如果你要先练自己的案例，可以读 [用 Codex 准备作品集页面素材](/blog/codex-portfolio-page-assets)。如果要逐项核对素材，配合 [作品集素材检查清单](/blog/codex-portfolio-page-assets-checklist) 更稳。

## 适合谁

适合已经能做基本信息整理、文案改写、截图打码、案例结构设计的新手。你不一定要会做完整网站，但要能问清客户想展示什么、哪些素材能公开、哪些结果有证据、哪些地方不能写太满。

也适合想做“低风险内容型项目”的人。相比直接接复杂开发，素材整理更偏流程和表达，但它仍然需要隐私意识和事实核对。

## 不适合谁

不适合客户完全不给素材，却要求你凭空写真实案例的任务。没有项目背景、截图、链接、职责说明和授权信息，就只能做模板或结构建议，不能写成真实交付。

也不适合要求你处理敏感客户项目、内部系统截图、未公开产品路线图或真实用户数据，但不给明确授权的任务。此类任务需要先暂停确认。

## 风险提醒

这类项目的主要风险有四个。第一是授权风险：客户自己是否有权展示素材也需要确认。第二是真实性风险：案例内容不能捏造。第三是范围风险：素材整理、页面设计、网站开发、SEO 和客户转化文案是不同任务。第四是效果风险：作品集能提升展示质量，但不能承诺一定带来订单。

报价前要把服务拆清楚。比如“整理 3 个案例素材包”和“上线一个作品集网站”不是同一件事；“改写客户提供文案”和“从零采访并写完整案例”也不是同一件事。

## 具体步骤

### 1. 判断客户材料是否够用

至少需要：项目名称或匿名名、项目类型、客户角色、原始问题、你的职责、结果证据、可公开素材、不能公开内容和目标读者。如果客户只有一句“帮我包装一下经历”，先做需求澄清，不要直接报价完整页面。

可以用 [Proposal 生成器](/tools/proposal-generator)生成确认问题，再人工改短。

### 2. 拆服务范围

基础范围可以是素材整理：把客户提供的素材分组、打标签、列缺口。进阶范围可以是案例文案：写背景、过程、结果、复盘。更大范围才是页面制作：布局、组件、发布、域名和维护。

如果涉及域名、部署平台、客户账号或付费服务，需要列为客户确认或明日待办，不要自行注册或授权。

### 3. 做小样验证

正式项目前，可以先整理一个案例小样：标题、背景、职责、证据、限制和 CTA。小样能帮助客户确认风格，也能帮助你判断素材是否足够。

如果小样阶段就发现客户要求虚构数据、夸大效果或展示未授权内容，应及时调整边界。

### 4. 设计交付物

轻量交付可以是一份素材表和缺口清单。中等交付可以是 2 到 3 个案例文案草稿。完整交付可以是作品集页面结构、文案、截图处理建议和发布前检查表。每一档都要写清修改轮次和不包含内容。

报价时可以用 [报价计算器](/tools/pricing-calculator)估算阅读素材、沟通、整理、改写和复核时间。

## 可以考虑接的任务

客户提供 3 个练习项目或公开项目链接，希望你帮他整理成案例页面草稿，并允许你标注哪些是 demo、哪些是真实任务。这类任务边界比较清楚，适合新手练习。

客户已有作品集页面，但案例描述太散，希望你整理标题、问题背景、职责和 CTA。这类任务也可以接，但要说明不包含设计重做和网站开发，除非另行报价。

## 应该暂停的任务

客户要求你把未完成项目写成成功案例，或要求隐藏真实来源。这类任务不适合继续。客户要求展示第三方品牌、客户后台、订单数据或内部截图，但不能提供授权，也应暂停。

还有一种情况是客户把作品集整理和“保证接到客户”绑定在一起。你可以提供素材整理和展示建议，但不能承诺结果。
