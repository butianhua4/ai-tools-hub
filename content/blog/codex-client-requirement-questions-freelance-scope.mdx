# Codex 需求不清楚的任务接单前怎么判断能不能做

需求不清楚时，新手最容易把“客户愿意聊”误认为“任务可以接”。其实接单前真正要判断的是：目标是否清楚、材料是否足够、验收是否可验证、权限是否安全、风险是否在你能负责的范围内。

这篇是草稿，只做接单范围判断参考，不构成报价承诺。建议和 [需求澄清主流程](/blog/codex-client-requirement-questions)、[检查清单](/blog/codex-client-requirement-questions-checklist)、[报价计算器](/tools/pricing-calculator) 一起使用。

## 适合谁

适合准备接 Codex 小单的新手，尤其是客户需求只有一两句话时，你不知道该继续问、先报价，还是直接放弃。本文帮助你把模糊任务分成可评估、待补充和应暂停三类。

也适合已经写过 Proposal、但经常后续返工的人。很多返工不是技术不够，而是一开始没有把验收、材料和边界问清楚。

## 不适合谁

不适合客户要求你在没有授权的情况下登录后台、修改生产配置、处理真实数据、接触支付或账号权限的任务。需求再诱人，也不能把安全边界交给侥幸。

也不适合没有任何验证方式的任务。如果客户无法说明如何判断完成，你就很难承诺交付。

## 风险提醒

客户需求不清楚时，不要用低价和快速承诺换成交。越是模糊，越要把报价前提写清楚。涉及注册、登录、授权、生产环境、高权限操作的事项，先记录为客户待确认，不要替客户完成。

## 具体步骤：三类判断

第一类，可以继续评估。客户能提供目标、材料、验收标准和测试环境，任务范围集中在页面、文案、样式、轻量脚本、构建报错、文档整理或可复现小问题。这类任务可以进入报价，但仍要写清未覆盖范围。

第二类，先补充材料。客户目标大致清楚，但缺链接、截图、仓库、报错、验收标准或环境说明。这时可以回复问题，不要直接报价。必要时可以提出“先做诊断，再报价修复”。

第三类，暂停。任务涉及生产数据、支付、账号权限、安全问题、不可回滚操作，或者客户拒绝说明授权来源。这类任务不适合新手直接接，可以只做需求整理和风险清单。

## 接单前的判断问题

1. 我能否用一句话复述客户目标？
2. 我是否知道客户会如何验收？
3. 客户是否给了足够材料让我判断工作量？
4. 是否能在测试环境或本地环境验证？
5. 是否涉及真实用户、支付、账号权限、密钥或生产环境？
6. 是否有明确截止时间和预算范围？
7. 是否能写出不包含哪些内容？
8. 如果客户马上要求开始，我是否仍然知道第一步做什么？

如果这些问题答不上来，就先不报价。可以先发送澄清问题，让客户补充。

## 报价前提模板

```text
This estimate is based on the following assumptions:
- The task can be verified in a test or local environment.
- No production credentials or sensitive user data are required.
- The scope is limited to the issue described above.
- Any login, authorization, payment, or admin changes will be confirmed by you first.
- If new requirements appear, I will confirm the updated scope before continuing.
```

中文也可以写：

```text
本次报价基于以下前提：
- 使用测试环境或脱敏材料评估；
- 不接收真实生产密钥或用户数据；
- 只处理当前描述的问题；
- 涉及登录、授权、支付、后台权限的操作由客户确认；
- 新增需求需要重新确认范围。
```

## 最小可接版本

如果完整任务不清楚，但客户愿意补充材料，你可以只接一个诊断阶段：整理需求、确认验收标准、列出缺失材料、识别风险、给出下一步建议。这个范围更安全，也能帮助你判断后续是否继续。

诊断阶段也要交付记录：客户目标、已知材料、缺失材料、风险点、建议范围、下一步待确认。不要把它包装成完整修复。

## 什么时候可以升级到正式执行

诊断阶段之后，只有当目标、材料、验收、权限和验证方式都清楚时，才适合进入正式执行。这里的“清楚”不是你自己感觉懂了，而是客户已经明确确认，并且你能把范围写成可交付条目。

如果客户补充材料后发现风险更高，比如从普通页面修改变成生产后台权限调整，应该重新评估，而不是沿用最初的报价。范围变化不是小事，它会影响时间、责任、验证方式和是否需要更有经验的人参与。
