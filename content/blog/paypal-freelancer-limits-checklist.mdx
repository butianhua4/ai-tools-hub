# PayPal 收款有哪些常见限制：新手检查清单

这份清单用于 PayPal 收款前自查。完整说明可以看 [PayPal 收款有哪些常见限制](/blog/paypal-freelancer-limits)。清单的重点不是马上发付款链接，而是先确认账户、平台、客户、费用和争议边界。

本文保持草稿状态，不替你注册 PayPal、不替你绑定银行账户、不替你判断税务或法律责任。任何真实注册、身份验证、提现、退款和争议处理，都应在明日人工复核后再执行。

## 适合谁

适合准备接海外小单、客户直接付款、模板定制或轻量咨询服务的新手。你可能已经被客户问到 PayPal，但还没有确认它是否适合当前订单。

也适合正在比较 PayPal、Payoneer、Wise 或平台内置收款的人。你可以用它先找出不确定项，再决定明天查哪些官方资料。

## 不适合谁

不适合正在处理大额交易、公司财务、税务申报、复杂合同、敏感商品或高争议风险订单的人。这些事项需要专业复核。

也不适合想用 PayPal 替代报价、合同、发票、验收和交付记录的人。收款工具只是入口，证据链仍然要自己保存。

## 具体步骤

1. 先完成订单来源检查。
2. 再完成账户与地区检查。
3. 继续完成付款、费用、争议和提现记录检查。
4. 把无法确认的问题写入明日人工复核清单。
5. 在复核完成前，不发送真实付款链接，不提交不必要敏感资料。

## 1. 订单来源检查

- [ ] 订单来自自由职业平台。
- [ ] 订单来自客户直接付款。
- [ ] 订单来自模板、数字产品或下载页面。
- [ ] 订单还不存在，只是在提前准备。
- [ ] 我知道平台是否要求使用平台内置收款。

如果订单来自平台，先看平台规则。如果订单还不存在，不急着注册或绑定账户，先完善服务说明和报价流程。

## 2. 账户与地区检查

- [ ] 我准备查 PayPal 当前是否支持我的地区和业务场景。
- [ ] 我准备查个人账户、商业账户或其他账户类型差异。
- [ ] 我准备查是否需要身份验证、地址证明、银行绑定或额外资料。
- [ ] 我准备核对姓名、地址、银行账户和平台资料是否需要一致。
- [ ] 我不会用他人资料或无法解释的账户信息处理收款。

## 3. 付款和费用检查

- [ ] 我准备查当前手续费、汇率、提现费用和可能的退款成本。
- [ ] 我知道客户付款币种、金额、地区和付款理由。
- [ ] 我准备在报价中单独记录费用影响，而不是事后才发现到账金额不同。
- [ ] 我知道费用截图和二手教程可能过期。
- [ ] 我不会把一次测试付款当作长期规则。

## 4. 争议和退款检查

- [ ] 我已经写清服务范围、交付物、修改次数和验收方式。
- [ ] 我已经写清退款边界和取消条件。
- [ ] 我准备保存客户确认、聊天记录、订单编号和交付文件。
- [ ] 我知道争议发生时需要证据，而不是只靠口头记忆。
- [ ] 如果客户要求模糊、金额异常或时间过急，我会先暂停。

## 5. 提现和记录检查

- [ ] 我准备查提现到本地银行的路径、时间和限制。
- [ ] 我准备保存付款状态、提现记录、汇率记录和平台通知。
- [ ] 我准备按订单整理收入，而不是只看余额变化。
- [ ] 我知道税务、发票和会计记录需要人工复核。
- [ ] 我不会把 PayPal 余额当作完整收入记录。

## 风险提醒

不要因为客户说“PayPal 很方便”就直接收款。先确认订单范围、付款理由、客户身份、交付证据和争议边界。

不要为了绕开平台规则而使用 PayPal。如果客户来自已有平台，应先看平台允许的收款方式。

不要忽视退款和争议。小额订单也可能出现误解、撤回或投诉，付款前把交付和退款写清楚更重要。

## 6. 明日人工复核清单

- [ ] PayPal 当前地区支持、账户类型、身份验证和银行绑定要求。
- [ ] 当前手续费、汇率、提现费用、退款和争议规则。
- [ ] 目标平台是否允许使用 PayPal 收款或提现。
- [ ] 是否需要税务资料、发票、合同或消费者提示。
- [ ] 当前订单是否适合 PayPal，还是更适合平台内置收款、Payoneer、Wise 或其他方式。
- [ ] 是否应该暂缓注册或暂缓发送付款链接。
