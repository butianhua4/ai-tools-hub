# Codex 怎么做第一个落地页：新手检查清单

如果你准备让 Codex 帮你做第一个落地页，先不要急着让它改代码。更稳的做法是按清单走一遍：先确认主题和素材，再让 Codex 提结构，最后小范围修改、预览、构建、记录。这样做出来的页面不一定复杂，但会更可控。

这份清单只适合新手练习和作品集准备，不代表你已经能独立接复杂客户项目。AI 可以辅助，但页面内容、风险判断、平台规则和最终交付仍然需要人负责。

## 适合谁

适合想用 Codex、Next.js、Vercel 做第一个静态页面的新手。你可以还不熟悉代码，但愿意一步一步确认页面目标、文案、模块、按钮和验证结果。

也适合正在准备作品集的人。一个小落地页如果能清楚展示你的流程，比一个复制来的“高级页面”更可信。

## 不适合谁

不适合想直接做客户生产项目、接支付、做后台、接入会员系统或承诺转化结果的人。也不适合想批量生成页面、批量发布、批量投标的人。练习落地页应该真实、低风险、可验证。

## 清单一：主题是否足够小

- 页面主题能用一句话说清楚。
- 页面只服务一个目标用户。
- 页面只有一个主要 CTA。
- 页面不承诺收入结果或接单结果。
- 页面不需要登录、支付、数据库和后台。
- 页面不使用别人未经授权的图片、案例或文案。

如果这几项做不到，就先缩小主题。比如把“AI 接单平台”改成“AI 接单每日执行表下载页”，范围会清楚很多。

## 清单二：素材是否准备好

- H1 标题已经写好。
- 副标题已经写好。
- 适合谁和不适合谁已经写好。
- 至少准备 3 个功能点或内容点。
- CTA 文案已经确定。
- 风险提醒已经写好。
- 如果有图片，确认是自己制作、授权使用或不使用图片。

素材越清楚，Codex 越不容易乱猜。不要把所有事情都交给 AI 编。

## 清单三：给 Codex 的指令是否安全

你可以先这样问：

```text
请先不要修改文件。帮我检查这个落地页需求是否适合新手练习：
主题：[填写主题]
目标用户：[填写用户]
CTA：[填写 CTA]
请输出页面结构、需要补充的素材、风险提醒和不建议做的复杂功能。
```

确认后再让它小范围改：

```text
现在只修改一个页面或一个组件，不要接支付、登录、会员系统、数据库或后端。
请保持页面简洁，适合手机访问，并在修改后告诉我需要运行哪些命令验证。
```

## 具体步骤

1. 先把页面主题缩小到一个明确用途，例如模板下载页、工具介绍页或作品集入口。
2. 写好 H1、副标题、适合谁、不适合谁和 CTA，不要让 Codex 凭空编业务目标。
3. 让 Codex 先检查结构，不要直接修改文件。
4. 确认结构后，只让它改一个页面或一个组件。
5. 修改后运行 `git status` 和 `npm run build`，记录改动和输出。
6. 用手机宽度、平板宽度和桌面宽度检查页面是否正常。
7. 保存截图、提示词和复盘，作为作品集练习记录。

## 清单四：页面结构是否完整

- 有清楚的 H1。
- 有一句能说明页面用途的副标题。
- 有“适合谁”。
- 有“不适合谁”。
- 有 3-5 个核心模块。
- 有一个主要 CTA。
- 有风险提醒或免责声明。
- 有相关内链，例如模板、工具或联系页。

新手常见问题是只做一个漂亮 Hero，却没有解释页面到底给谁用、下一步是什么。落地页不是海报，它需要引导行动。

## 清单五：修改后如何验证

修改后至少做这些检查：

```bash
git status
npm run build
```

如果能本地预览，再检查手机宽度、平板宽度和桌面宽度。重点看标题是否换行正常，按钮是否可点，卡片是否挤在一起，长中文是否溢出。

如果构建失败，不要为了通过而随便删文件、关 ESLint 或忽略 TypeScript 错误。把错误复制到 [报错解释器](/tools/error-explainer)，先弄清楚含义。

## 清单六：作品集记录怎么写

练习完成后，记录这些内容：

- 原始目标：我想做什么页面。
- 使用工具：Codex、Next.js、Tailwind、Vercel 等。
- 我的提示词：我怎么让 Codex 帮忙。
- 修改范围：改了哪些页面或组件。
- 验证结果：构建是否通过、截图是否正常。
- 风险边界：没有接支付、没有客户数据、没有夸大承诺。
- 下一步：还想改进什么。

这个记录未来可以变成作品集说明。客户或自己回看时，不只是看到页面，还能看到你的思考过程。

## 常见错误

第一个错误是主题太大，导致 Codex 生成一堆你看不懂的结构。第二个错误是 CTA 太多，页面看起来很忙但没有重点。第三个错误是文案太夸张，容易变成不可信的营销页。第四个错误是不检查移动端，结果手机上一堆重叠。第五个错误是没有保存过程，做完以后说不清自己做了什么。

## 风险提醒

不要复制别人落地页的整段文案，不要抓取别人图片，不要伪造客户案例，不要写收入保证类、绝对安全类或暗示不用人工审核的表达。AI 辅助内容必须人工审核，尤其是 CTA、收益表达、工具推荐和平台规则相关内容。

如果以后想把这个练习页面用于真实服务引流，要重新核对文案、隐私政策、免责声明和联系渠道。练习页可以是开始，但不能跳过审核。

## 总结

第一个落地页不需要很大，它需要清楚、真实、能验证。按主题、素材、指令、结构、验证、记录这六个清单走一遍，你会比直接让 Codex “帮我做个网站”稳很多。

慢一点没关系。能说明页面为什么这样做，能证明构建通过，能保存移动端截图，就是新手阶段很扎实的一步。
