# npm: command not found 怎么解决？4 个原因（没装 Node / PATH / nvm）

终端里敲 `npm` 报 `npm: command not found`（Windows 上是 `'npm' 不是内部或外部命令`），意思是系统在 PATH 里找不到 npm 这个可执行文件。**npm 是跟着 Node.js 一起装的**，所以问题基本就 4 类：没装 Node、装了没进 PATH、用 nvm 没激活、或终端没重开。

## 第一步：确认 Node 到底装没装

```bash
node -v
npm -v
```
- 两个都报 not found → Node 根本没装（见 ①）
- `node -v` 有、`npm -v` 没有 → 少见，重装 Node 即可
- 刚装完就报 → 多半是 PATH 或终端没重开（见 ②③）

## ① 根本没装 Node.js（最常见）

**修复**：去 nodejs.org 装 LTS 版，或用版本管理器（更推荐，见 ③）。装完**新开一个终端**再试 `node -v`。别下载来历不明的安装包。

## ② 装了但 PATH 没生效（命令找不到）

**成因**：Node 装了，但它的目录没加进 PATH，或当前终端还是装之前打开的。

**修复**：
- 先**完全关掉终端重新打开**（PATH 改动对已开的终端不生效）——这一步能解决一大半。
- 还不行，确认安装目录在 PATH 里：macOS/Linux 看 `echo $PATH`；Windows 在「系统环境变量」里确认有 Node 的路径。

## ③ 用了 nvm 但没激活 / 没选版本

**成因**：装了 nvm（Node 版本管理器）但没 `nvm use` 选版本，或 nvm 的初始化脚本没加进 shell 配置。

**修复**：
```bash
nvm install --lts
nvm use --lts
node -v        # 确认生效
```
macOS/Linux 确认 `~/.zshrc` 或 `~/.bashrc` 里有 nvm 的加载行；改完 `source ~/.zshrc` 或重开终端。

## ④ Windows 特有：装完没重启终端 / 用错终端

**成因**：Windows 装完 Node 后，旧的 CMD/PowerShell 窗口拿不到新 PATH。

**修复**：关掉所有终端窗口重新打开；用管理员 PowerShell 试；必要时注销重登一次让环境变量全量生效。

## 推荐：直接用 nvm 管理 Node

与其反复折腾 PATH，建议一开始就用 nvm（macOS/Linux）或 nvm-windows 管理 Node 版本，既能避免权限和 PATH 问题，也方便不同项目切版本（配合 `.nvmrc`）。
