# 新手练习项目记录表怎么用

练习项目的价值不只在于“做出来”，还在于你能不能复盘自己怎么做出来。对 AI 接单新手来说，记录表能把一次小练习变成可展示、可复查、可继续改进的作品集素材。

这篇仍是草稿。发布前需要人工补充真实记录样例和截图，确认没有客户信息、账号、密钥或未授权素材。

## 适合谁

适合正在做第一个网页、CSS 修复、报错排查、部署练习或 Proposal 练习的新手。你可能还没有真实客户项目，但可以先把每次练习记录得像一次小交付。

也适合用 Codex、ChatGPT 或 Claude Code 辅助学习的人。工具可以帮你推进，但记录表让你知道哪些是自己理解的，哪些还需要复核。

## 不适合谁

不适合只想堆数量、不复盘过程的人。也不适合把虚构练习写成真实客户交付的人。

如果练习材料来自客户、课程、公司仓库或付费模板，要先确认是否允许记录和公开。不能把私有内容直接放进作品集。

## 记录表要写什么

一份可用的练习记录至少包括：

1. 练习目标：今天要解决什么问题。
2. 输入材料：需求、报错、截图、相关文件。
3. 使用工具：Codex、浏览器、Git、构建命令等。
4. 实际改动：改了哪些文件或文案。
5. 验证方式：运行了什么命令，看了哪些页面。
6. 结果截图：前后对比或关键页面。
7. 风险和限制：哪些没验证、哪些需要人工复核。
8. 下一步：明天继续做什么。

记录越具体，后面写作品集和 Proposal 越省力。

## 具体步骤

1. 开始练习前先写目标，不要边做边改方向。
2. 保存原始需求或报错，不要只保存 AI 总结。
3. 每完成一个小改动，就用 Git 或截图记录变化。
4. 运行项目检查，例如 lint、build、测试或手动页面验证。
5. 把失败也写下来，失败记录同样能证明你会排查。
6. 练习结束后写 3 句话总结：做了什么、怎么验证、下次改什么。
7. 如果涉及授权、注册、平台连接或客户确认，放进明日待办。

## 可复制模板

```text
练习项目记录表

日期：
练习主题：
目标：
输入材料：
使用工具：
关键步骤：
改动文件：
运行命令：
验证结果：
截图位置：
遇到的问题：
需要人工复核：
明日待办：
```

这份模板可以放在 Notion、Markdown、表格或纸质笔记里。形式不重要，关键是每次都能补完整。

## 如何变成作品集素材

不是每条练习记录都适合公开。先从记录里挑出最清楚、最安全、最能展示流程的一条，再改写成案例：背景、任务、过程、结果、截图、反思。

练习项目要明确标注为练习。不要把模拟客户、AI 生成需求或课程项目写成真实商业客户。真实比漂亮更重要。

## 一个记录示例

例如你练习修复一个移动端按钮换行问题，可以这样记录：目标是让按钮在 390px 宽度下保持可读；输入材料是截图和页面路径；使用工具是浏览器开发者工具、Codex 和 Git；关键改动是调整按钮宽度、间距和换行规则；验证方式是桌面、手机宽度截图和 `npm run build`。

复盘时不要只写“成功”。可以写：“这次学会了先定位父级布局，再修改按钮样式；还没有验证旧浏览器；明天补一张不同宽度截图。”这样的记录未来可以变成作品集案例，也能提醒自己哪里还没掌握。

如果练习失败，也要记录失败原因。比如依赖安装没有通过、看不懂报错、截图无法复现，这些都是下一轮练习的入口。

## 风险提醒

不要记录或公开密码、token、客户资料、后台截图、订单数据、私有仓库链接和未授权素材。不要用记录表承诺收入或成交结果。不要把没有验证的 AI 输出写成已完成交付。

如果需要注册平台、连接 GitHub、绑定部署服务或申请客户授权，先写入明日待办，确认后再继续。

## 明日待办

- 补一份真实练习项目记录样例。
- 做截图脱敏检查。
- 核对练习素材来源和授权。
- 挑一条记录改写成作品集草稿，但先保持不公开。
