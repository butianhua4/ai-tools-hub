# SEO 工具新手先用哪些免费功能

新手做 SEO 检查，不需要一开始购买复杂工具。先用免费功能把基础问题看清楚：页面能不能被访问，标题和描述是否清楚，sitemap 和 robots 是否正常，链接有没有断，移动端是否能读，页面速度有没有明显问题，内容有没有过度承诺。

SEO 工具只能提供线索，不能保证排名、流量、成交或客户增长。对 AI 接单新手来说，免费工具更适合做“检查证据”，而不是拿来承诺结果。

## 适合谁

适合刚开始做个人网站、作品集、模板下载页、工具页或客户小站检查的新手。你可能还不会完整 SEO 策略，但可以按步骤把基础错误查出来。

也适合准备把 SEO 检查做成练习项目的人。比如为一个本地 demo 页面检查标题、描述、链接、移动端截图和构建结果，最后写成作品集案例。

## 不适合谁

不适合需要大型站点增长策略、复杂关键词研究、竞品监控、程序化 SEO、国际化 SEO 或企业级技术 SEO 的场景。那些任务需要更多数据、权限和经验。

也不适合向客户承诺排名的人。任何工具都不能替你保证搜索结果位置。新手可以提供检查、修复建议和证据，但不能承诺外部结果。

## 风险提醒

不要把免费工具截图包装成“已经优化成功”。比如 Lighthouse 分数、索引状态、关键词建议和页面速度只是某一时刻的检查结果，不代表搜索引擎一定会给流量。

如果工具需要登录、站点验证、接入 Search Console、读取 Analytics、访问客户后台或安装脚本，先写入明日待办并等待授权。没有客户确认前，不要添加跟踪代码、修改 DNS、上传验证文件或接触生产环境。

## 具体步骤

按这个顺序使用免费功能：

1. 用浏览器打开页面，确认页面可访问。
2. 查看标题、描述和正文是否匹配搜索意图。
3. 检查 canonical、robots 和 sitemap 是否明显异常。
4. 检查页面内部链接和重要外链。
5. 用移动端视口查看文字是否可读。
6. 用免费速度工具或浏览器检查性能问题。
7. 保存检查截图、命令输出或页面快照。
8. 写出限制：未接入真实搜索数据、未验证站点所有权、未做长期排名跟踪。
9. 把需要登录、授权或客户确认的事项列入明日待办。

这个流程更像体检，不是承诺疗效。检查越具体，越适合新手练习。

## 浏览器自查

最先用的工具就是浏览器。打开页面后检查：

- 标题是否说明页面主题
- 首屏是否能看懂
- 主要按钮是否可点击
- 文字是否被遮挡
- 图片是否加载
- 页面是否有明显报错
- 移动端是否需要横向滚动

这一步不用账号，也不会改动网站。适合所有练习项目。

## 页面源码和元信息

下一步检查标题、描述、canonical 和语言信息。可以用浏览器查看页面源代码，也可以用站内脚本或构建输出检查。

新手重点看：

- 每个页面是否有唯一标题
- 描述是否具体
- canonical 是否指向正确 URL
- 页面是否误加 noindex
- 草稿是否没有进入 sitemap
- sitemap 是否包含公开页面

如果你在客户站点上发现问题，先截图和记录，不要直接改生产环境。

## Sitemap 和 Robots

免费检查通常包括打开：

```text
/sitemap.xml
/robots.txt
```

看它们是否能访问，是否指向正确域名，是否明显拦截了不该拦截的页面。不要为了测试随意修改 robots，因为它会影响搜索引擎抓取。

本站类似检查可以参考 [SEO 可搜索性审计](/docs/seo-searchability-audit) 的思路，但真实项目仍要按当前站点配置人工确认。

## 链接和内容检查

SEO 工具能发现一些链接问题，但新手也可以手动检查：

- 重要内链是否能打开
- CTA 链接是否指向正确页面
- 外链是否仍可访问
- 内容是否和标题一致
- 是否有重复或空洞段落
- 是否有夸大承诺
- 是否有未授权素材或客户信息

如果页面涉及收款、法律、税务、医疗、安全或平台规则，必须加人工复核，不要只靠工具提示。

## 性能和移动端

免费性能工具适合找明显问题，例如图片过大、布局跳动、移动端按钮太小、首屏加载慢。它们不适合直接拿来承诺排名。

建议保存两类证据：

- 检查截图
- 你做了哪些具体修改

比如“压缩首页图片”“减少无用脚本”“修复移动端溢出”。这比写“SEO 优化完成”更可信。

## 明日待办

以下事项需要授权或真实后台，先记录：

- 登录 Google Search Console
- 验证站点所有权
- 查看搜索查询和点击数据
- 接入或查看 Analytics
- 修改 DNS 或上传验证文件
- 修改生产站点 robots
- 添加跟踪代码
- 访问客户 CMS 后台

这些事项没有授权前，不要操作。草稿里只写检查计划和待确认问题。

## 可以复制的练习记录

```text
SEO practice check:
- Page checked:
- Title:
- Description:
- Sitemap status:
- Robots status:
- Mobile screenshot:
- Link issues:
- Performance notes:

- Verification:
- Limits: no Search Console access, no long-term ranking data, no client backend access.
```

这段适合放进练习项目记录。重点是把限制写出来。
