# Codex 怎么做第一个落地页：接单前怎么判断能不能做

如果你已经能用 Codex 做一个练习落地页，下一步很容易冒出一个念头：这种项目是不是可以去 Upwork 或 Fiverr 接单？先给结论：可以研究，但不要急着投。落地页看起来简单，实际包含需求理解、文案边界、视觉还原、移动端适配、部署方式、客户验收和后续修改。

Codex 可以帮你提高制作效率，但它不能替你判断项目是否适合新手。你需要先看范围是否局部、素材是否完整、结果是否可验证、风险是否可控。本文给的是保守判断框架，不承诺成交，不鼓励自动投标，也不建议新手碰不清楚的生产项目。

## 适合谁

适合已经做过 1-3 个自己的练习页面，能用 Codex 修改文案、卡片、按钮、简单样式，并且愿意运行 `npm run build` 做基本验证的新手。你不需要成为资深开发者，但至少要能说明页面做了什么、改了哪些文件、如何查看结果。

也适合正在筛选小项目的人。你可能看到客户写着 landing page、simple website、one page、CSS fix、responsive page，这些词看起来友好，但仍然要拆清楚范围。

## 不适合谁

不适合第一天刚接触 Codex，就想直接接客户项目的人。也不适合看不懂需求、不敢问客户、不愿保存截图、不愿测试移动端的人。落地页不是只要生成代码就结束，客户通常会看细节：标题、间距、表单、按钮、图片、手机端、加载速度。

如果项目包含支付、登录、会员、后台、生产数据库、复杂追踪脚本、广告投放承诺或客户敏感信息，新手不应该独立接。可以学习这些词是什么意思，但不要把高风险任务包装成练手小单。

## 三档判断：练习、可尝试、先避开

第一档，只适合练习：你还不能独立运行项目，或者只能看 Codex 的回答，不能验证结果。这时应该继续做自己的模板页、工具介绍页、作品集入口页，不要对客户承诺。

第二档，可以小心尝试：客户只需要一个静态单页，素材明确，文案已提供，不接支付，不接后端，不要求复杂动画，验收标准是页面能打开、手机端正常、按钮链接正确。你仍然应该先问清楚范围，再保守报价。

第三档，先避开：客户要求完整品牌重设计、复杂转化优化、A/B 测试、广告追踪、支付接入、会员系统、多语言、后台管理、生产站迁移，或者要求你直接在生产环境改。这里面任何一个点都可能超过新手能力边界。

## 接单前必须确认的材料

- 客户是否已有文案，还是要你从零写。
- 是否有 logo、品牌色、图片或示例页面。
- 页面是一个静态页，还是包含表单、支付、登录或后台。
- 是否需要部署到 Vercel、Netlify 或客户自己的主机。
- 是否需要连接 Google Analytics、Search Console 或广告追踪。
- 移动端验收标准是什么。
- 客户是否接受先做一个小范围版本，再确认是否追加。

这些问题问清楚，才能判断它是练习级落地页，还是复杂商业页面。新手最怕不是不会做，而是不知道自己接了什么。

## 具体步骤

1. 保存客户原始需求，不要只保存自己的理解。
2. 标出页面类型：模板下载页、工具介绍页、作品集页、服务介绍页，还是广告投放页。
3. 检查是否涉及支付、登录、后台、生产数据或复杂追踪。
4. 要求客户提供文案、素材、参考页面和验收标准。
5. 用自己的练习项目做一个类似结构的小 demo。
6. 运行 `npm run build`，检查移动端截图。
7. 再决定是否投标、如何报价、如何写清边界。

## 可以复制的客户确认模板

```text
Thanks for sharing the landing page request. Before I confirm the quote, I need to clarify a few details:
1. Is this a static one-page landing page, or does it need forms, payment, login, or backend features?
2. Do you already have the copy, logo, images, and brand colors?
3. What should be considered finished for desktop and mobile?
4. Where should the page be deployed?
5. Are there any tracking scripts or platform rules I should know about?

I will only confirm a limited scope after reviewing these details.
```

这段话的重点是“limited scope”。新手接小单时，最重要的是限制范围，而不是显得什么都能做。

## 常见错误

第一个错误是把 landing page 当成“简单网页”。有些落地页只是静态介绍页，有些却和广告投放、数据追踪、支付转化绑定在一起。名字一样，风险完全不同。

第二个错误是客户不给素材，你就让 Codex 全部编。AI 可以帮你整理草稿，但商业页面的标题、卖点、案例和图片必须真实，不能虚构。

第三个错误是只看桌面端。落地页经常来自手机访问，移动端按钮、卡片、表单和图片必须单独检查。

第四个错误是报价前不问部署。只做页面和负责上线是两种工作量。如果还要绑定域名、配置环境变量或处理构建错误，需要单独确认。

## 风险提醒

不要承诺具体收入、转化率、广告效果或平台审核结果。不要复制客户竞争对手页面，不要抓取别人图片，不要使用未经授权的案例。不要接受绕开 Upwork、Fiverr 等平台规则的要求。

如果客户要求平台外私下交易、免费完成完整页面测试、直接给你生产账号权限，先暂停。你可以问清楚，也可以拒绝；跳过高风险项目不是失败，而是在保护账号和信誉。

## 总结

Codex 可以帮助新手做落地页，但接单前要先判断范围。静态、素材完整、验收清楚、低风险的单页可以作为未来小项目方向；涉及支付、后台、生产数据、广告承诺和复杂追踪的项目，应该先避开或找人复核。

真正稳的第一步，不是急着投标，而是把自己的练习页面做成可验证案例：有截图、有构建结果、有复盘。这样以后判断项目时才有底气。
