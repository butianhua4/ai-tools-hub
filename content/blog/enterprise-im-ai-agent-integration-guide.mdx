# 企业微信、飞书、Slack 怎么接 AI Agent：消息入口、权限和人工接管

很多企业真正使用 AI，不是从一个独立网页开始，而是从企业微信、飞书、Slack 这样的聊天入口开始。员工在群里提问，AI 查知识库、生成摘要、创建工单、提醒负责人，体验更自然。但 IM 接入也更容易失控：群聊里谁能触发、AI 能看到什么、能不能调用工具、回答错了谁接管，都要提前设计。

本文是待复核草稿。Agent 权限可以看 [Agent 工具权限控制](/blog/agent-tool-permission-safety-guide)，人工审核可以看 [Agent 人工审核流程](/blog/agent-human-review-loop-guide)。

## 适合谁

适合做企业内部助手、客服协作助手、销售跟进助手、项目日报助手、知识库问答机器人和自动化通知的人。只要用户已经在 IM 工具里工作，接入聊天入口就很自然。

也适合接企业 AI 自动化项目的人。客户常说“能不能接到企业微信或飞书”，这背后其实包含身份、权限、知识库、消息格式和人工接管。

## 不适合谁

不适合没有权限设计就把 Agent 拉进所有群。不同群的资料、角色和操作权限不同。

也不适合让 Agent 在群里自动执行高影响动作。创建草稿可以，发送正式通知、改数据、触发付款、修改权限要谨慎。

## 消息入口

消息入口可以是私聊、群聊、关键词触发、斜杠命令、按钮或表单。私聊适合个人任务，群聊适合团队协作，按钮和表单适合固定流程。

不要让 Agent 对所有消息都响应。最好设计明确触发方式，例如 @机器人、命令、固定按钮或特定频道。

## 用户身份和权限

Agent 要知道是谁在提问、属于哪个团队、有哪些权限。知识库回答和工具调用都要按用户身份过滤。

如果群聊里多人可见，回答内容也要考虑群权限。某个员工有权限查看的资料，不代表整个群都有权限看到。

## 知识库和工具

企业 IM Agent 常见能力包括查制度、查产品文档、总结会议、创建工单、发送提醒、查询 CRM、生成日报。每个能力都要分清只读和写入。

知识库回答要带来源，工具调用要有日志。高影响操作要在执行前确认。

## 人工接管

AI 回答不确定、资料不足、用户情绪明显、涉及客户投诉或高风险内容时，应转人工。转人工时要带上用户问题、上下文、AI 已尝试内容和建议处理方式。

不要只发一句“请人工处理”。信息越完整，接管越顺。

## 常见错误

第一个错误是让 Agent 在群里过度发言。企业 IM 里噪音很容易让用户反感。

第二个错误是没有区分私聊和群聊权限。群聊回答可能泄露本不该公开的信息。

第三个错误是没有记录消息和工具调用。后续复盘和责任归属会困难。

## 交付检查

企业 IM Agent 交付时，建议按“入口、权限、能力、兜底”四部分验收。入口包括私聊、群聊、命令和按钮；权限包括用户身份、群权限和资料范围；能力包括知识库、摘要、工单和提醒；兜底包括低置信度回答、转人工和错误日志。

还要准备几类真实测试：普通员工提问、无权限员工提问、群里多人追问、资料不足、AI 需要转人工、工具调用失败。企业 IM 场景比网页更嘈杂，测试越贴近日常聊天，后续越稳。

## 风险提醒

企业聊天工具里可能包含客户资料、合同信息、员工信息和内部决策。Agent 接入前要确认数据边界和日志策略。

涉及对外承诺、报价、法律、财务、人事和客户投诉时，应加入人工审核或转人工机制。

## 具体步骤

第一步，确定接入平台和触发方式。第二步，设计用户身份和群权限。第三步，选择知识库和工具能力。第四步，为高影响动作加入确认。第五步，记录消息和工具调用日志。第六步，设计人工接管流程。需要接入清单，可以从 [工具导航](/tools) 下载或联系人工协助设计。
