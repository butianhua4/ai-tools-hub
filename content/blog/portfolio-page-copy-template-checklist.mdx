# 作品集页面文案模板怎么写真实：新手检查清单

作品集页面写完后，不要马上发布。新手最需要检查的不是页面够不够花哨，而是内容是否真实、截图是否安全、能力边界是否清楚、客户是否能知道下一步要给你什么材料。

这篇仍是草稿。发布前需要人工补充截图示例和脱敏规则，并核对平台、素材和客户授权边界。

## 适合谁

适合已经写好作品集页面初稿，但不确定能不能公开的新手。你可能有练习项目、模拟案例、AI 辅助记录或少量真实交付，需要逐项检查风险。

也适合准备把作品集放到个人网站、Notion、GitHub Pages 或其他平台的人。上线前的复核，比上线后再删敏感内容更稳。

## 不适合谁

不适合想用虚构项目包装经验的人。也不适合没有授权就公开客户项目、截图、业务名称、聊天记录和后台数据的人。

如果页面涉及商业客户、私有代码、收费项目或用户数据，必须先取得明确授权或做匿名化处理。

## 检查清单

发布前逐项确认：

1. 每个案例是否真实存在，练习项目是否标明是练习。
2. 是否写清楚你负责的部分，而不是把团队或 AI 输出全写成自己的能力。
3. 截图里是否出现邮箱、订单、token、后台路径、客户名称或真实用户数据。
4. 图片、图标、字体、模板是否有授权或可公开使用。
5. 是否写了不接的范围，例如支付、生产数据库、安全漏洞、敏感账号。
6. CTA 是否具体，例如“发送页面链接和截图”，而不是泛泛地说“联系我”。
7. 手机端和桌面端是否都能读完，按钮和文字是否重叠。
8. 是否保留了内部复核记录，方便以后解释案例来源。

## 具体步骤

1. 打开作品集页面初稿，逐段标出“事实”“判断”“承诺”。
2. 删除或改写无法证明的承诺。
3. 给每个案例补充目标、过程、结果、验证方式。
4. 检查所有截图和链接，遮盖敏感信息。
5. 把 AI 辅助部分写成流程，不要写成独立能力夸口。
6. 请另一位人工复核文案是否夸大。
7. 确认无误后再进入发布或预览。

## 可复制的发布前记录

```text
作品集页面发布前检查

页面链接：
案例数量：
练习案例：
真实客户案例：
是否有客户授权：
截图是否脱敏：
素材来源：
能力边界：
不接范围：
CTA：
需要明日处理：
```

如果某一项填不出来，就先不要公开。空白项通常代表风险没有被看见。

## 常见错误

第一个错误是把“参与过”写成“独立完成”。第二个错误是把 AI 生成页面当成真实客户结果。第三个错误是只写工具名，不写验证方法。第四个错误是没有说明自己暂时不接什么任务。

真实的作品集可以很朴素，但必须经得起追问。客户问“你怎么验证的”，你要能拿出截图、命令或交付记录。

## 复核时怎么改文案

复核时可以把夸大的句子改成可验证句子。比如“我能快速完成各种网站”可以改成“我目前主要处理静态页面文案、简单样式问题和部署前检查”。“熟悉所有 AI 工具”可以改成“会用 Codex 辅助读代码，并用 git diff 和构建命令检查输出”。

如果案例来自练习项目，就明确写“练习项目”。如果案例来自客户项目，但没有公开授权，就只保留内部复盘，不要放到公开页面。作品集不是越多越好，而是每一条都能解释来源、过程和限制。

最后再看 CTA 是否要求客户提供具体材料。比如“请发送页面链接、截图、预期效果和截止时间”比“欢迎咨询”更适合新手接低风险任务。

## 风险提醒

不要公开客户隐私、项目后台、私有仓库、报价记录、合同截图或聊天记录。不要承诺结果性指标。不要使用未授权素材。不要把平台规则不允许公开的内容放进作品集。

需要注册域名、绑定部署平台、申请客户授权或连接 GitHub 的事项，先放进明日待办。

## 明日待办

- 做一版截图脱敏示例。
- 人工核对所有案例和素材来源。
- 准备客户授权确认模板。
- 检查作品集页面移动端显示。
