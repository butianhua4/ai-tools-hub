# Notion 管理接单流程怎么用

Notion 可以帮新手把接单流程整理清楚，但它不是自动接单工具，也不能替你判断项目风险。更合适的用法是：用一个轻量数据库记录线索、需求、报价、交付、收款状态和复盘，让每个项目都有可追踪的下一步。

这篇是草稿，只做学习和流程设计参考。它不要求你现在注册或授权 Notion，不建议导入客户敏感资料，也不提供法律、隐私或合同结论。真实共享权限、客户资料字段、自动化集成和团队协作设置，都放进明日待办，等人工复核后再处理。

## 适合谁

适合正在准备接第一个 AI 辅助小项目的新手。你可能已经在用 ChatGPT、Codex、Claude Code、Vercel 或 GitHub 做练习，但客户线索、报价、交付记录和复盘散落在聊天记录里。

也适合已经有几个模拟项目或作品集练习的人。你可以先用 Notion 建一个私人项目表，把每次练习记录成案例，而不是只保存一堆截图和临时文件。

## 不适合谁

不适合想让 Notion 替自己自动回复客户、自动报价、自动承诺结果的人。Notion 适合记录和提醒，不适合替你承担客户沟通和交付责任。

也不适合直接保存客户密码、生产数据库、支付资料、身份证明、私密合同或高权限链接。新手流程表应先记录低敏摘要，需要保存敏感资料时必须另做权限和合规复核。

## 一个够用的项目表

新手不需要一开始做复杂系统。一个 Notion 数据库可以先包含这些字段：

| 字段 | 用途 |
| --- | --- |
| 项目名称 | 一句话描述任务 |
| 来源 | Upwork、Fiverr、朋友、个人网站、练习项目 |
| 状态 | 线索、待确认、已报价、进行中、待验收、已完成、已放弃 |
| 风险等级 | 低、中、高 |
| 下一步 | 当前最小动作 |
| 截止时间 | 需要回复或交付的时间 |
| 交付物 | 页面、文档、代码、截图、报告或模板 |
| 证据链接 | 需求、报价、交付记录和客户确认 |
| 复盘 | 做完后记录问题和改进 |

字段越少，越容易坚持使用。不要把第一个工作流做成复杂仪表盘，否则维护成本会盖过它的价值。

## 具体步骤

1. 新建一个私人 Notion 页面，命名为“接单项目记录”。
2. 建一个表格数据库，只添加项目名称、来源、状态、风险等级、下一步、截止时间和复盘。
3. 把已有练习项目录入 3 条，不录入客户敏感信息。
4. 给每个项目写一句“客户要什么”和一句“我下一步做什么”。
5. 为真实客户项目添加报价、交付物、验收方式和沟通记录链接。
6. 每天只看“下一步”和“截止时间”，不要沉迷整理页面样式。
7. 项目结束后写复盘：哪里卡住、如何验证、下次怎么缩小范围。

## 推荐状态流

线索表示只是看到需求，还没有判断是否适合。待确认表示需要问客户更多问题。已报价表示范围和价格已经发出，但还没有开始。进行中表示你正在做低风险交付。待验收表示已经发给客户确认。已完成表示结果和记录都保存好。已放弃表示风险过高、需求不清或不适合新手。

这个状态流的重点是“暂停也有记录”。很多新手只记录成功项目，结果无法复盘为什么某些项目不该接。把已放弃原因写清楚，反而能提高后续判断质量。

## 风险提醒

不要把 Notion 页面公开给客户，除非你已经确认共享范围、权限和可见字段。一个错误的分享链接，可能暴露其他项目、客户姓名、报价或内部复盘。

不要把客户敏感信息直接放进工作流表。能用摘要就不用原文，能用链接就不要复制全文，能不保存就不保存。

不要把自动化集成作为第一步。Notion API、Zapier、Make、表单同步和邮件提醒都涉及授权和数据流向，应该等流程稳定后再评估。
